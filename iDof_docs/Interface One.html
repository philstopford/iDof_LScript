About the Interface
iDof’s interface has been redesigned completely.  Everything is exactly what it says, and nothing unneeded is visible.
 
Quick tip about the center and far point objects.  If, before you add iDof Channels, you create nulls in your scene called “center_point_IDOF” and “far_point_IDOF”, the Channels plugin will automatically assign these appropriately when you add the Image Filter.  You will still need to set up other options in order to see the results in the render.  This was a ‘hidden function’ added to streamline what could be the slowest part of the setup.  To speed this up further, you could create a Macro in your Lightwave Layout that adds these nulls automatically.
Interface: iDof Channels
Matte Type:
   Choose between the Greyscale matte (which gives independent channel output options) and the RGB matte (new, which writes a color depth of field matte to all the color channels).
Write to RGB Channels:
   Writes the new color depth matte to the all the color channels.  This option is only available with the RGB matte type.
Write to Red Channel:
   Writes the depth matte generated with this instance of iDof Channels to the red buffer. This option is only available with the Greyscale matte type.
Write to Green Channel:
   Same as red, only to the green channel.
Write to Blue Channel:
   Same as red and green, only to the blue channel.
Write to Alpha Channel:
   Same as above, only to the alpha channel.
Write to TGA Sequence:
   This will write the depth matte generated by iDof Channels to a 32bit TGA image sequence on disk.  Note that F9 renders will also output to disk with this option activated.  iDof cannot differentiate between F9 and F10 renders.
Prefix:
   With the TGA Sequence option activated, you use this to select the Save Location on disc and Filename Prefix.  Please note that you are only specifying the Prefix here.  iDof Channels will enumerate and add the .tga extension for you at rendertime.  If no location is selected, iDof Channels will write the images to the Content directory root.
Camera:
   Define the render camera here.  The black point of the depth matte is based on this camera setting, so choosing a camera other than the render camera could yield unpredictable results.
Center point (previously Black point):
   The object you select here will be the center area of the depth matte.  With a Greyscale matte, this will be the black area.  With an RGB matte, it’s the green area.
Far point (previously White point):
   The object you select here will be the end of the fade from black to white in the iDof Greyscale depth matte (and the end of the fade from green to red/blue in the RGB depth matte).
Spread:
   This pushes the center area further toward the far point before the fade begins.  100% is a smooth fade from the center point to the far point.  50% is halfway between the center point and far point before the fade begins.  0% is a hard line at the far point.

Limitations
iDof is still based on the depth buffer, so transparencies and volumetrics will still render improperly.  This is a common problem with DOF as a post process in general.
